Overview

What We Built

The AUDD Checkout SDK is an open-source toolkit that lets any web merchant accept AUDD payments on Solana with a single script tag and two lines of code. It handles everything from Solana Pay URL generation and QR rendering to on-chain payment detection and confirmation — with zero dependencies in the browser bundle.

The problem it solves: AUDD is a fully regulated, 1:1 AUD-backed stablecoin with real utility — but there is no easy, copy-paste integration path for merchants. Stripe-style simplicity has not existed for AUDD. This SDK fills that gap.

The project is live, open-source, and fully functional. The GitHub repository includes the browser widget, TypeScript SDK, and an interactive demo storefront.

Impact

Why This Matters for AUDD

AUDD's own website lists merchant payments and remittance as primary use cases. Yet today, a developer who wants to accept AUDD on their site must manually construct Solana Pay URLs, handle SPL token validation, and build their own polling infrastructure. That friction kills adoption.

This SDK removes every step between "I want to accept AUDD" and "AUDD is in my wallet." It is the missing distribution layer — equivalent to what Stripe.js did for card payments, but for AUD on-chain.

  • Works with every major Solana wallet — Phantom, Solflare, Backpack
  • Solana settlement in under 2 seconds at ~$0.001 per transaction
  • Merchant receives AUDD directly — no custodian, no intermediary
  • Fully open-source — any developer can fork, extend, or integrate

The SDK generates a unique reference key per payment session, encodes it into a Solana Pay URL, and polls findReference() to detect the transaction on-chain. Once detected, validateTransfer() confirms the correct recipient, AUDD token mint, and exact amount before firing the success callback.
